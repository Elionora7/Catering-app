# Catering App - Complete Project Summary

## 📋 Project Overview

A full-stack Next.js 14+ TypeScript catering application with comprehensive features for meal ordering, event prebooking, admin management, and customer interactions. Built with modern web technologies and best practices.

---

## 🛠️ Technology Stack

### Core Framework & Language
- **Next.js 14.1.0** - React framework with App Router
- **TypeScript 5** - Type-safe development
- **React 18.2.0** - UI library
- **React DOM 18.2.0** - React rendering

### Styling & UI
- **Tailwind CSS 3.3.0** - Utility-first CSS framework
- **PostCSS 8** - CSS processing
- **Autoprefixer 10.0.1** - CSS vendor prefixing

### State Management & Data Fetching
- **@tanstack/react-query 5.17.9** - Server state management
- **React Context API** - Client-side state (Cart)

### Authentication & Authorization
- **NextAuth.js 4.24.5** - Authentication framework
- **bcryptjs 2.4.3** - Password hashing
- **JWT Sessions** - Stateless authentication

### Database & ORM
- **PostgreSQL** - Relational database (via Docker)
- **Prisma 5.7.1** - Type-safe ORM
- **@prisma/client 5.7.1** - Prisma client

### Validation
- **Zod 3.22.4** - Schema validation

### Additional Features
- **react-qr-code 2.0.12** - QR code generation
- **qrcode 1.5.3** - QR code utilities

### Development Tools
- **tsx 4.7.0** - TypeScript execution
- **ESLint 8** - Code linting
- **eslint-config-next 14.1.0** - Next.js ESLint config

## 🗄️ Database Schema

### Models

#### User
- `id` (String, UUID, Primary Key)
- `email` (String, Unique)
- `name` (String)
- `password` (String, Hashed)
- `role` (Enum: ADMIN, CUSTOMER)
- `createdAt` (DateTime)
- `updatedAt` (DateTime)
- Relations: `orders`, `prebookings`

#### Meal
- `id` (String, UUID, Primary Key)
- `name` (String)
- `description` (String, Optional)
- `price` (Decimal)
- `imageUrl` (String, Optional)
- `category` (String, Optional)
- `isAvailable` (Boolean)
- `createdAt` (DateTime)
- `updatedAt` (DateTime)
- Relations: `orderItems`

#### Event
- `id` (String, UUID, Primary Key)
- `name` (String)
- `description` (String, Optional)
- `date` (DateTime)
- `location` (String, Optional)
- `maxGuests` (Int, Optional)
- `createdAt` (DateTime)
- `updatedAt` (DateTime)
- Relations: `prebookings`

#### Order
- `id` (String, UUID, Primary Key)
- `userId` (String, Foreign Key)
- `status` (Enum: PENDING, CONFIRMED, PREPARING, READY, DELIVERED, CANCELLED)
- `totalAmount` (Decimal)
- `createdAt` (DateTime)
- `updatedAt` (DateTime)
- Relations: `user`, `items`

#### OrderItem
- `id` (String, UUID, Primary Key)
- `orderId` (String, Foreign Key)
- `mealId` (String, Foreign Key)
- `quantity` (Int)
- `price` (Decimal)
- Relations: `order`, `meal`

#### Prebooking
- `id` (String, UUID, Primary Key)
- `userId` (String, Foreign Key)
- `eventId` (String, Foreign Key)
- `guestCount` (Int)
- `status` (Enum: PENDING, CONFIRMED, CANCELLED)
- `specialRequests` (String, Optional)
- `createdAt` (DateTime)
- `updatedAt` (DateTime)
- Relations: `user`, `event`

## 🔐 Authentication & Authorization

### NextAuth Configuration
- **Provider**: Credentials (Email/Password)
- **Session Strategy**: JWT
- **Session Duration**: 30 days
- **Password Hashing**: bcryptjs (10 rounds)
- **Custom Pages**: `/auth/login`, `/auth/register`

### Role-Based Access Control
- **ADMIN**: Full access to admin panel, all CRUD operations
- **CUSTOMER**: Access to menu, cart, orders, prebookings

### Protected Routes
- `/admin/*` - Admin-only routes with layout-level protection
- API routes use `requireAuth()` and `requireAdmin()` helpers

---

## 📡 API Endpoints

### Authentication
- `POST /api/auth/register` - User registration
- `POST /api/auth/[...nextauth]` - NextAuth handler (login, session)

### Meals
- `GET /api/meals` - List all meals
- `POST /api/meals` - Create meal (Admin only)
- `GET /api/meals/[id]` - Get single meal
- `PUT /api/meals/[id]` - Update meal (Admin only)
- `DELETE /api/meals/[id]` - Delete meal (Admin only)

### Events
- `GET /api/events` - List all events (with prebooking summary)
- `POST /api/events` - Create event (Admin only)
- `GET /api/events/[id]` - Get single event (with prebookings)
- `PUT /api/events/[id]` - Update event (Admin only)
- `DELETE /api/events/[id]` - Delete event (Admin only)

### Orders
- `GET /api/orders` - List orders (filtered by user role)
- `POST /api/orders` - Create order (Authenticated)

### Prebookings
- `GET /api/prebooking` - List prebookings (filtered by user role)
- `POST /api/prebooking` - Create prebooking (Authenticated)
- `GET /api/prebooking/[id]` - Get single prebooking
- `PUT /api/prebooking/[id]` - Update prebooking (Owner/Admin)
- `DELETE /api/prebooking/[id]` - Delete prebooking (Owner/Admin)

### Menu QR
- `GET /api/menu/qr` - Generate menu QR code data

## 🚀 Getting Started

1. **Install Dependencies**
   ```bash
   npm install
   ```

2. **Set Up Environment**
   ```bash
   cp .env.example .env
   # Edit .env with your values
   ```

3. **Start Database**
   ```bash
   docker-compose up -d postgres
   ```

4. **Generate Prisma Client**
   ```bash
   npm run db:generate
   ```

5. **Run Migrations**
   ```bash
   npm run db:migrate
   ```

6. **Seed Database** (Optional)
   ```bash
   npm run db:seed
   ```

7. **Start Development Server**
   ```bash
   npm run dev
   ```

8. **Open Browser**
   ```
   http://localhost:3000
   ```

- **Event**: Name, date, optional description/location/maxGuests
- **Order**: Array of items (mealId, quantity)
- **Prebooking**: EventId, guestCount, optional specialRequests
